Why MAIN world for WebGazer?為什麼 WebGazer 要在 MAIN world 執行？

WebGazer v2.1.2 uses TF.js which calls new Function() for WebGL shaders. MV3 extension isolated-world CSP blocks unsafe-eval. Running in MAIN world bypasses this restriction. Gaze coordinates are bridged back to isolated world via DOM CustomEvent (no object refs cross the boundary).

WebGazer v2.1.2 使用 TF.js，後者在 WebGL shader 中呼叫 new Function()。MV3 擴充功能的 isolated world CSP 封鎖了 unsafe-eval。在 MAIN world 執行可以繞過這個限制。視線座標透過 DOM CustomEvent 橋接回 isolated world（不傳遞物件參照）。

🔴 CRT ABI Mismatch — PyInstaller Frozen Exe CrashCRT ABI 不相容 — PyInstaller 凍結執行檔崩潰

Root cause: openvino_genai.dll is compiled with MSVC v14.40+ which uses SRWLOCK-based std::mutex. PyInstaller bundles msvcp140.dll v14.31 which uses CRITICAL_SECTION layout. When the frozen exe loads the bundled CRT, the inlined mutex code in openvino_genai.dll reads NULL at offset 8 → crash at mtx_do_lock+0x9c.

Fix: Drop exact-named CRT DLLs from the PyInstaller bundle so the exe falls back to System32's v14.44. Applied in both native_host.spec and benchmark.spec. Numpy's private msvcp140-<hash>.dll is preserved (exact-name match avoids it).

根本原因：openvino_genai.dll 使用 MSVC v14.40+ 編譯，其 std::mutex 基於 SRWLOCK。PyInstaller 打包的 msvcp140.dll 為 v14.31，使用 CRITICAL_SECTION 版面配置。凍結執行檔載入打包的 CRT 後，openvino_genai.dll 的 mutex 內嵌程式碼在 offset 8 讀到 NULL → 崩潰於 mtx_do_lock+0x9c。

修正方式：從 PyInstaller bundle 中精確刪除 CRT DLL，讓執行檔回退使用 System32 的 v14.44。已套用至 native_host.spec 與 benchmark.spec。Numpy 的私有 msvcp140-<hash>.dll 保留（精確名稱匹配排除之）。

# In benchmark.spec / native_host.spec# 在 benchmark.spec / native_host.spec 中
_CRT_EXACT = {
    'msvcp140.dll', 'msvcp140_atomic_wait.dll',
    'vcruntime140.dll', 'vcruntime140_1.dll',
    'vccorlib140.dll', 'concrt140.dll',
}
def _is_crt(name):
    b = os.path.basename(name).lower()
    return b in _CRT_EXACT or b.startswith('api-ms-win-crt-')

🟡 Qwen Chat Template Causes Early EOSQwen Chat Template 導致提前 EOS

Using Qwen2.5's <|im_start|>/<|im_end|> chat template causes the model to emit EOS after the first bullet point only, producing incomplete summaries.

Solution: Use raw completion prompts seeded with 1. — the model outputs a full numbered list, which is then converted to • bullets by _format_output().

使用 Qwen2.5 的 <|im_start|>/<|im_end|> chat template 會使模型在第一條重點後立即輸出 EOS，造成摘要不完整。

解決方式：改用以 1. 為引子的 raw completion prompt，模型會輸出完整編號清單，再由 _format_output() 轉換為 • 條列格式。

🔵 WebGazer CSP Issue — Must Run in MAIN WorldWebGazer CSP 問題 — 必須在 MAIN World 執行

WebGazer v3.x uses MediaPipe which dynamically loads JS — blocked by page CSP. WebGazer v2.1.2 uses TF.js which calls new Function() for WebGL shaders — blocked by MV3 isolated-world CSP (unsafe-eval not allowed).

Solution: Run webgazer.js in MAIN world (where page CSP applies, not extension CSP). Bridge gaze coordinates to isolated world via DOM CustomEvent carrying {x, y} viewport coords only.

WebGazer v3.x 使用 MediaPipe，需動態載入 JS，被頁面 CSP 封鎖。WebGazer v2.1.2 使用 TF.js，其 WebGL shader 呼叫 new Function()，被 MV3 isolated world CSP 封鎖（不允許 unsafe-eval）。

解決方式：在 MAIN world 執行 webgazer.js（適用頁面 CSP，不受 extension CSP 限制）。透過 DOM CustomEvent（只傳遞 {x, y} 座標）橋接至 isolated world。

🟢 Language Detection & Traditional Chinese Conversion語言偵測與正體中文轉換

Language is detected per-paragraph by CJK character ratio (not the page lang attribute). If more than 8% of characters are CJK → 'zh'; else 'en'.

Qwen2.5 outputs Simplified Chinese even when instructed to use Traditional. Post-processing with zhconv.convert(text, 'zh-tw') converts simplified → traditional. Also strips markdown bold markers (**text**) emitted by Qwen.

語言依每段落的 CJK 字元比例偵測（非頁面 lang 屬性）。超過 8% 為 CJK → 'zh'；否則 'en'。

Qwen2.5 即使被指示輸出正體中文，仍會輸出簡體。後處理使用 zhconv.convert(text, 'zh-tw') 轉換簡體→正體，同時移除 Qwen 輸出的 Markdown 粗體標記（**text**）。

# In content.js: per-paragraph language detection# 在 content.js：每段落語言偵測
function detectTextLang(text) {
    const cjk = (text.match(/[　-鿿豈-﫿]/g) || []).length;
    return cjk / text.length > 0.08 ? 'zh' : 'en';
}

# In native_host.py: output post-processing# 在 native_host.py：輸出後處理
def _format_output(raw, lang):
    text = raw.strip()
    if lang == 'zh':
        import zhconv
        text = zhconv.convert(text, 'zh-tw')          # simplified → traditional# 簡體→正體
    text = re.sub(r'\*{1,2}([^*\n]+)\*{1,2}', r'\1', text)  # strip **bold**# 移除 **粗體**
    text = re.sub(r'(?m)^\s*(?:\d+[.、）)]\s*|[-*]\s+)', '• ', text)  # → bullets# → 條列
    return text.strip()
