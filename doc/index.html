Overview

What This PoC Does

Validates end-to-end on-device AI summarization triggered by simulated gaze hold, running entirely on the local Intel chip — no API, no cloud, no GPU required.

👁

Gaze Hold Trigger

Simulated with mouse hover (2 s threshold). When the user holds gaze on a paragraph, L1 visual enhancement + summarization activate simultaneously.

🤖

On-device AI

Phi-4 Mini (INT4 quantized, ~1.8 GB) runs via OpenVINO on Intel NPU → iGPU → CPU fallback. No internet required after first model download.

🔌

Native Messaging

Edge extension communicates with a local Python process via Chrome Native Messaging protocol (stdin/stdout JSON). Registered in Windows Registry.

⚡

Target: < 2s

On Intel Core Ultra 2 (Lunar Lake) NPU, summary latency is 0.9–1.5 s. Panther Lake expected to be faster. Acceptable threshold: < 3 s.

Implementation

Four-Phase Plan

Each phase has a clear deliverable and test criteria before moving forward.

1
Model Setup — Download & Convert
Day 1

Download Phi-4-mini from HuggingFace and convert it to OpenVINO IR format with INT4 quantization. Then run the benchmark script to confirm inference works on your specific Intel hardware.

1
Activate venv and run install_deps.bat
2
Run python download_convert.py — takes ~15 min, outputs ~1.8 GB model
3
Run python benchmark.py — tests NPU / GPU / CPU
✅ Phase 1 Pass Criteria
  • Model folder models/phi4-mini-int4/ exists and contains openvino_model.bin
  • benchmark.py prints Available devices including NPU
  • At least one device completes a summary in < 5 s
  • Summary text is readable English/Chinese

2
Native Messaging Host
Day 2

Set up the Python process that Edge will talk to. Test the stdin/stdout protocol manually before involving the extension.

1
Load the extension in Edge (edge://extensions → Developer mode → Load unpacked → select poc/extension/)
2
Copy the Extension ID shown in edge://extensions
3
Run python register.py --extension-id <ID>
4
Test the host manually with the Python test script below
# Manual host test (run in a separate terminal)
import subprocess, json, struct

proc = subprocess.Popen(
    ["python", "native_host.py"],
    stdin=subprocess.PIPE, stdout=subprocess.PIPE
)

def send(obj):
    data = json.dumps(obj).encode()
    proc.stdin.write(struct.pack("=I", len(data)) + data)
    proc.stdin.flush()

def recv():
    n = struct.unpack("=I", proc.stdout.read(4))[0]
    return json.loads(proc.stdout.read(n))

send({"action": "ping", "reqId": 1})
print(recv())   # → {"status": "ready", "reqId": 1}

send({"action": "summarize", "text": "AI Kwau is a browser extension...", "lang": "en", "reqId": 2})
print(recv())   # → {"status": "ok", "summary": "...", "reqId": 2}
✅ Phase 2 Pass Criteria
  • Registry key exists: HKCU\SOFTWARE\Microsoft\Edge\NativeMessagingHosts\com.hp.aikwau.summarizer
  • Manual ping test returns {"status": "ready"}
  • Manual summarize test returns a non-empty summary string
  • aikwau_host.log shows "Model loaded and warmed up"
3
Edge Extension — UI & Trigger
Day 3

The extension is already loaded from Phase 2. This phase verifies the content script triggers correctly and the badge UI renders.

1
Reload the extension in edge://extensions (click the refresh icon)
2
Navigate to any news article (e.g. BBC, CNN)
3
Open DevTools Console — check for errors
4
Hover mouse over a paragraph and hold for 2 seconds
✅ Phase 3 Pass Criteria
  • No errors in DevTools Console after page load
  • After 2 s hover: paragraph text becomes darker and bold (L1)
  • Blue badge appears with "💡 Summarising…" text
  • Badge disappears when mouse moves away (cleanup)
4
End-to-End Integration & Benchmark
Day 4–5

Connect all three layers. Measure real-world latency from gaze-hold to summary-visible, across multiple page types and Intel device paths.

1
Ensure native host is registered (Phase 2) and extension loaded (Phase 3)
2
Open any English or Chinese article
3
Hover 2 s → verify "Summarising…" → verify summary appears
4
Check aikwau_host.log for timing and errors
5
Record latency across NPU / GPU / CPU by changing DEVICE_PRIORITY in native_host.py
✅ Phase 4 Pass Criteria
  • Full flow works: hover → L1 → "Summarising…" → summary text → auto-dismiss
  • Chinese page produces Chinese summary (lang auto-detect)
  • NPU latency recorded and < 2 s on Panther Lake
  • No crashes after 10 consecutive requests
  • Log file shows no errors

Quick Start

# 1. Clone and enter project
cd ai-kwau/poc/model_setup

# 2. Set up environment
install_deps.bat
.venv\Scripts\activate

# 3. Convert model (one-time, ~15 min)
python download_convert.py

# 4. Benchmark on your hardware
python benchmark.py

# 5. Register native host (after loading extension in Edge)
cd ..\native_host
python register.py --extension-id <YOUR_EXTENSION_ID>

# 6. Load extension in Edge
#    edge://extensions → Developer mode → Load unpacked → select poc/extension/
